# Idento Kiosk Desktop v2 — дизайн

**Дата:** 2026-07-21
**Статус:** утверждён (объём, sidecar+standalone, GitHub Releases, rewrite с нуля, общая UI-библиотека — решения пользователя)
**Источник дизайна:** Claude Design, проект `165a9ba5-4bb1-4ede-9048-546ccb1742af`, файл `Idento Kiosk.dc.html` (канва, фиксация turn 4) + логотипы из `design_handoff_idento_logo/assets`.

## 1. Цель и объём

Переписать десктоп-приложение `desktop/` (Tauri 2) под дизайн-систему «kiosk-strict» из канвы: полный флоу — pre-flight визард, staffed-run (две компоновки), self-service оболочка, полноэкранные вердикты, деградированные состояния, адаптация 1920×1080 → 1366×768 → 1024×600 (Pi-тачскрин). Плюс: агент оборудования как sidecar + standalone-дистрибуция, автообновления через GitHub Releases.

**Вне объёма (следующие итерации):** светлая тема (1b), офлайн-очередь отметок (в экране «сервер недоступен» зарезервировано место под «N отметок ждут синхронизации»), подпись кода Windows/macOS (нотаризация), обновление standalone-агента поверх systemd (v1 — переустановка скриптом).

## 2. Зафиксированные дизайн-решения канвы

- Статус-система **1e «чипы с подписями»**: зелёная тишина → янтарный чип с текстом проблемы (работа продолжается) → красный баннер с действием (линия стоит). Замена тостов; статусы постоянны.
- Staffed-компоновка — **настройка станции**: 1a (статус-полоса сверху, 76px) или 1c (боковая панель оператора, для 24″+; статусы с деталями — пинг, версия, порт; счётчик и лог не исчезают под вердиктом — цветом говорит только рабочая зона).
- **Self-service — отдельная оболочка**: attract-экран (брендслот 380×130, медленный дрейф 60 с от выгорания, переключатель языка — единственная интерактивная зона), приватные вердикты (только имя, автосброс 8 с), полный локдаун, выход по staff-QR.
- **Вердикты полноэкранные**, обратная связь ≤200 мс (поле цвета сразу, детали догружаются), имя ≥96px на 1920 (шкала 116→84→64), цвет всегда дублирован иконкой и подписью. Красный только для отказа и блокировки станции. «Уже отмечен» — без автовозврата, ждёт решения оператора; «код не найден» — нейтральный тёмный + подсказка формата («это EAN товара»); отказ (аннулирован) — красный.
- **Аппаратный сканер — основной ввод**: без рамки-видоискателя; метафора штрихкода + «луч» (transform-only анимация), явная живость канала («Сканер COM3 · слушает» дышит), диагностика «последний принятый код и его результат».
- **Отметка и печать — раздельные исходы**: успех отметки не отменяется проблемой печати; очередь недопечатанных бейджей копится и допечатывается после восстановления принтера.
- **Pre-flight** — линейный хребет из 5 шагов, цель < 3 мин, один активный шаг, прогресс виден всегда.
- Деградации — «честная блокировка вместо спиннера», каждая — спроектированный экран с «что делать», всегда двухходовка: починить причину или обойти другим устройством.
- Масштабные ступени: заголовок idle 62→48→38, имя вердикта 116→84→64, статус-полоса 76→64→56 px; на 1024×600 лог убирается (экран мал для трёх зон).

## 3. Архитектура

### 3.1 Воркспейсы и пакеты

- Корневой `package.json`: `workspaces: ["packages/*", "panel", "desktop"]` — desktop входит в общий воркспейс (его отдельный `package-lock.json` удаляется, лок ведётся корневым).
- **`@idento/ui` получает subpath-экспорт `@idento/ui/kiosk`** (`./src/kiosk/index.ts`) и `@idento/ui/kiosk-theme.css`. Киоск-компоненты не тянут панельную тему; общие утилиты (`cn`, cva) шарятся. Панельные экспорты не меняются.
- `desktop/src` — только экраны, роутинг, стейт станции, клиенты API/агента.

### 3.2 Фронт desktop (rewrite с нуля)

Свежие мажоры на момент реализации: React 19, Vite, Tailwind 4, TypeScript strict, react-router 7, Zustand (стейт станции/здоровья), i18next RU/EN. Портируются с адаптацией: `lib/api.ts` (axios, 401→relogin, 403 tenant_suspended), `lib/agent.ts` (Tauri invoke-прокси `agent_request`), `lib/config.ts` (backend URL), QR-логин (jsqr), i18n-скелет. `sonner`-тосты в run-режиме **не используются** (статус-система вместо тостов); допустимы в pre-flight для второстепенного.

### 3.3 Rust/Tauri

Обновить crates; плагины: `tauri-plugin-shell` (sidecar), `tauri-plugin-updater`, `tauri-plugin-process` (relaunch после апдейта), `tauri-plugin-single-instance`. Сохраняется команда `agent_request` (проксирование HTTP к агенту без CORS). Новые команды: `agent_sidecar_status` при необходимости. Иконки: регенерация `tauri icon` из нового логотипа (assets из design_handoff), включая tray-template (закрывает pending-пункт логотип-роллаута для desktop).

Inter (400–800) бандлится локально (woff2, без CDN — киоск может быть офлайн). Типографика — CSS-переменные со ступенями по media/container-брейкпоинтам 1366 и 1024.

### 3.5 Компоненты `@idento/ui/kiosk`

`StatusChip`/`StatusChips` (узлы: сервер, агент, принтер, сканер; три уровня эскалации), `BlockingBanner` (красный, действие + авто-повтор с таймером), `TopStatusBar` (1a), `OperatorPanel` (1c), `VerdictScreen` (success / already / not-found / refused + self-вариант), `BarcodeBeam`, `RecentLog`, `KioskButton`/`KioskInput` (крупные touch-цели), `PreflightShell` (шаги + прогресс), `BrandSlot`, `LanguageToggle`.

### 3.6 Роутинг и стейт

- `/setup/connection` → `/setup/login` → `/setup/equipment` → `/setup/event` → `/setup/mode` (выбор self-service добавляет шаг настройки staff-QR выхода).
- `/run` — staffed (компоновка из настройки станции), `/self` — self-service (локдаун, выход по staff-QR).
- Стейт станции (Zustand + персист): backend URL, токен, событие, режим (`staffed-bar` | `staffed-panel` | `self`), устройства (принтер, сканер, ввод: serial|wedge|camera), брендинг attract, update-URL override.
- Health-store: поллинг сервера (лёгкий API-запрос), агента (`/health`), принтера/сканера (через агент); выводит уровень эскалации станции по правилам §2. Очередь печати: клиентская, retry после восстановления, счётчик «N бейджей ждут печати» в футере/панели.

### 3.7 Сканирование

Три источника, настройка станции: (1) serial-сканер через агент — поллинг `/scan/consume`; (2) keyboard-wedge (HID) — глобальный обработчик с буфером и таймингом; (3) камера (jsqr) — с экраном восстановления «камера запрещена». Формат-подсказки для чужих кодов (EAN и т.п.).

## 4. Агент: sidecar + standalone

- **Sidecar**: бинарь в `desktop/src-tauri/sidecars/idento-agent-<target-triple>`, `bundle.externalBin: ["sidecars/idento-agent"]`. Апп при старте: если настроен «встроенный агент» (по умолчанию) — spawn sidecar с портом из конфига, мониторинг `/health`, рестарт с экспоненциальным backoff, остановка при выходе. Если настроен «внешний агент» (URL) — только подключение.
- **Standalone**: те же Go-бинари (linux amd64/arm64, windows amd64, darwin arm64/amd64) прикладываются к GitHub Release; в `agent/dist/` — systemd-юнит, install-скрипт (headless Pi у принтера). Версия агента отображается в чек-листе оборудования (`GET /info`).
- Агент обновляется вместе с аппом (sidecar заменяется установщиком апдейта). Standalone v1 — переустановка скриптом.

## 5. Обновления ПО

- `tauri-plugin-updater`, артефакты подписываются minisign-ключом (`TAURI_SIGNING_PRIVATE_KEY`/`_PASSWORD` — GitHub secrets; публичный ключ в `tauri.conf.json`).
- Релиз по тегу `desktop-v*`: workflow-матрица — macOS arm64+x64, Windows x64, Linux x64 (deb+AppImage), Linux arm64 (deb+AppImage, ARM-раннер `ubuntu-24.04-arm`; если недоступен — best-effort job). Каждый job собирает Go-агент для своей цели, кладёт в sidecars, собирает бандлы, `createUpdaterArtifacts: true`; итог — GitHub Release с installers + `latest.json`.
- UX-политика: проверка при запуске и раз в сутки; run-режим никогда не прерывается — тихий чип «Доступно обновление X.Y.Z» только в pre-flight/настройках; установка по подтверждению оператора (download → install → relaunch). URL манифеста переопределяем в настройках станции (закрытые сети: зеркало или файловый шар).

## 6. Тестирование и CI

- Vitest + Testing Library: state-machine эскалации (юнит), print-queue retry, wedge-парсер, VerdictScreen (варианты), format-hint. Тесты киоск-компонентов — в `packages/ui` рядом с существующими.
- Дев-прогоны с `agent --mock`.
- CI PR-джоба desktop: `npm ci` (корень) → typecheck → lint → тесты → `vite build` (без Tauri-бандла). Tauri-бандл — только в релизном workflow.
- `packages/ui` ESLint-запрет нативных контролов распространяется и на kiosk-подпапку (свои контролы — уже правило пакета).

## 7. Риски и допущения

- ARM-раннеры GitHub для private-repo могут быть платными/недоступными — тогда Linux arm64 собирается в best-effort job (или self-hosted позже); остальная матрица не блокируется.
- Без подписи кода Windows покажет SmartScreen-предупреждение при первой ручной установке; апдейтер после установки работает по своей подписи. Задокументировать для операторов.
- Wedge-сканеры не требуют агента — сканер-чип в статусах для wedge-режима показывает «клавиатурный ввод» без liveness serial-канала.
- `desktop` в воркспейсе: Tauri CI должен ставить зависимости из корня репо (beforeBuildCommand учитывает cwd).
