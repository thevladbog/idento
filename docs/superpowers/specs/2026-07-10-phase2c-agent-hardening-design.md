# Фаза 2C: Хардненинг принтер-агента (AGENT-SEC-01/02/03) — дизайн

- **Дата:** 2026-07-10
- **Статус:** на утверждении
- **Тип:** cross-subsystem security (agent + desktop; web — конфиг без правок кода)

## Цель

Закрыть 3 Critical-находки принтер-агента из аудита: полное отсутствие
аутентификации (AGENT-SEC-01), прослушивание всех сетевых интерфейсов
`0.0.0.0` (AGENT-SEC-02), уязвимость к CSRF/relay через вредоносную веб-страницу
(AGENT-SEC-03) — минимально затрагивая клиентов.

## Модель угроз (из аудита)

Агент работает на машине оператора рядом с принтером/сканером и слушает
`http://:12345`. Сейчас любой запрос принимается без проверок. Угрозы:
1. **Сетевой сосед** — любое устройство в той же Wi-Fi может печатать, менять
   конфиг принтеров, открывать serial-порты (следствие bind на `0.0.0.0`).
2. **Вредоносная веб-страница** в браузере оператора — `fetch('http://localhost:12345/print', …)`
   (CSRF; `rs/cors` не защищает, т.к. не блокирует actual-запрос) и relay
   произвольных TCP-байт на любой IP:port через `/printers/add` + `/print`.

## Кто ходит в агент (факт из кода)

- **web** (`web/src/lib/agent.ts`): браузерный axios напрямую на `http://localhost:12345`.
- **desktop** (`desktop/src/lib/agent.ts`): в Tauri — через Rust-команду `agent_request`
  (`desktop/src-tauri/src/commands.rs`); в браузер-dev — fallback на localhost.

## Ключевые решения (утверждены)

1. **Полный фикс:** localhost-bind + shared-token + Origin/Content-Type проверка.
2. **web — Origin-only:** браузерный клиент не может держать файловый токен;
   защищается localhost-bind + Origin/Content-Type. Код web-аутентификации **не
   меняется**.
3. **desktop — token:** Tauri читает токен из файла агента и шлёт `Authorization: Bearer`.

## Дизайн

### Агент (Go) — основные изменения

**1. Bind только на localhost (AGENT-SEC-02).**
Слушать `127.0.0.1:<port>` вместо `:<port>`. Флаг/конфиг `--host` со значением по
умолчанию `127.0.0.1` (не `0.0.0.0`), чтобы осознанный оператор мог переопределить.

**2. Авторизация запроса (AGENT-SEC-01 + 03) — middleware перед всеми хендлерами.**
Запрос разрешён, если выполнены ВСЕ базовые условия и хотя бы один способ
аутентификации:
- **База (всегда):** `Host`-заголовок указывает на localhost (`127.0.0.1`/`localhost`
  + порт) — анти-DNS-rebind; для мутирующих (POST/DELETE) запросов
  `Content-Type: application/json` (блокирует simple-request CSRF, форсируя
  preflight, который мы можем отклонить).
- **Аутентификация (одно из):**
  - **токен:** заголовок `Authorization: Bearer <token>` совпадает с токеном
    агента (путь desktop);
  - **origin:** заголовок `Origin` входит в allowlist агента (путь web).
- Иначе — `401`/`403`, хендлер не выполняется.

`/health` — исключение (без авторизации), чтобы клиенты могли проверять
доступность.

**3. Токен агента.**
При старте: если в `~/.idento/agent_config.json` нет `auth_token` — сгенерировать
криптостойкий (32 байта, base64/hex) и сохранить (файл уже используется
loadConfig/saveConfig, права `0600`). Токен логируется один раз при старте (для
ручной диагностики), не при каждом запросе.

**4. Origin allowlist.**
Конфиг `allowed_origins` (в `agent_config.json` и/или env `AGENT_ALLOWED_ORIGINS`,
CSV). По умолчанию — origin(ы) web-приложения (напр. `http://localhost:5173` в dev
+ прод-домен) и desktop (tauri-схема). Пустой список = ни один origin не проходит
(остаётся только токен-путь).

**5. Заменить/убрать разрешающий `rs/cors` `*`.**
CORS-заголовки настроить согласованно с allowlist (echo the allowed Origin), НЕ
`*`. Но безопасность обеспечивает серверная проверка (п.2), а не CORS.

**6. Валидация relay-таргета (усиление AGENT-SEC-04/05, попутно).**
`/printers/add`/`/print` — ограничить IP/порт (например, запретить внешние/не-приватные
IP или добавить allowlist), не отправлять произвольные байты на любой адрес.
*(Если объёмно — вынести в backlog; основной фикс — auth + localhost.)*

### Desktop (Rust/Tauri) — минимальная правка

`agent_request` (`desktop/src-tauri/src/commands.rs`): прочитать `auth_token` из
`~/.idento/agent_config.json` и добавить заголовок `Authorization: Bearer <token>`
ко всем проксируемым запросам к агенту. Origin/Content-Type desktop-путь может не
использовать (аутентифицируется токеном), но `Content-Type: application/json` для
POST оставить.

### Web — без правок кода аутентификации

Браузер сам шлёт `Origin`. Нужно лишь, чтобы origin web-приложения был в allowlist
агента (конфиг агента, не код web). Проверить, что web шлёт `Content-Type:
application/json` на POST (axios по умолчанию — да).

## Верификация

- Агент: `go build ./...`, `gosec ./...` (Issues:0), `govulncheck` чист.
- **Тесты агента (первые в agent/ — репо не имеет тестов):** middleware отклоняет
  запрос без токена и без allowlist-origin (401); пропускает с валидным токеном;
  пропускает с allowlist-origin; отклоняет чужой origin; отклоняет не-localhost
  Host; `/health` доступен без auth.
- Ручная проверка потока: desktop печатает (токен), web печатает (origin), запрос
  с «чужой страницы»/curl без токена и origin — отклонён.
- desktop: `cargo build`, сборка фронтенда.

## Границы / не-цели

- Хеширование токена агента на диске (он и так локальный, права 0600) — не делаем.
- Полноценный TLS для localhost-агента — не нужен (loopback).
- Pairing-UX для web (ввод токена) — сознательно НЕ выбран; web на Origin-only.
- AGENT-BUG/QUAL находки (гонки, утечки портов, тесты) — отдельный трек.

## План работ

1. Спек (этот документ) → утверждение.
2. Implementation-план (writing-plans): агент (bind, middleware, токен, allowlist,
   тесты) → desktop (token header) → верификация потока.
3. Исполнение (subagent-driven) → PR.

## Риски

| Риск | Смягчение |
|---|---|
| Origin web-приложения не в allowlist → web не печатает | дефолтный allowlist включает dev+prod origin web; конфигурируемо |
| Desktop не находит токен (файл/права) | agent_request деградирует с понятной ошибкой; токен создаётся агентом при старте |
| Ломается dev-поток (web на 5173 → agent) | allowlist по умолчанию содержит localhost:5173; документируется |
| localhost-bind ломает сценарий «агент на другой машине» | флаг `--host` для осознанного оверрайда (по умолчанию безопасно) |
