# On-Prem Phase 2 — P2.2 Frontend Mode Awareness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hide the "Register" link on the login page and the "Super Admin" nav item, and actively redirect direct `/register` navigation to `/login`, when the backend reports `onprem` mode via the existing `GET /api/instance` endpoint.

**Architecture:** A new module-level memoized fetch (`web/src/lib/instance.ts`) backs a small React hook (`web/src/hooks/useInstanceMode.ts`), consumed by three independent UI call sites: `Login.tsx` (conditional render), `App.tsx` (a new `SaasOnlyRoute` guard mirroring the existing `ProtectedRoute`), and `Layout.tsx` (an added condition on the existing `is_super_admin` check). No backend changes — `GET /api/instance` already exists and returns everything needed.

**Tech Stack:** React 18.3.1 + TypeScript + react-router-dom + the existing `axios`-based `api` client (`web/src/lib/api.ts`) + vitest/@testing-library/react. No new npm dependencies.

## Global Constraints

- **Safe-default (allow-list) gating.** Every consumer starts from — and falls back to on fetch failure — `mode: 'onprem'`. Saas-only UI (the Register link, the `/register` route, the Super Admin nav item) is shown/enabled ONLY on a confirmed `mode === 'saas'` response. Never invert this (e.g. `mode !== 'onprem'` reads the same today but is the wrong default-on-failure direction — always compare `=== 'saas'`).
- **One shared, memoized fetch.** `getInstanceInfo()` in `web/src/lib/instance.ts` issues `GET /api/instance` at most once per page load, regardless of how many components call `useInstanceMode()`. Never fetch `/api/instance` directly from a component.
- **No new backend changes, no new i18n keys.** `GET /api/instance` (`{mode, version, license}`) already exists and is unauthenticated. The i18n keys `dontHaveAccount`, `register`, `superAdmin` already exist in both `en`/`ru` blocks in `web/src/i18n.ts` — this plan only changes when existing JSX renders, never its copy.
- **`/super-admin`'s own route guard (`ProtectedRoute requireSuperAdmin` in `App.tsx`) is NOT modified.** It already correctly gates on `user.is_super_admin` regardless of mode; this plan only adds a mode check to the *nav link's* visibility in `Layout.tsx`, not to route access control.
- **Frontend gate:** every task ends with `cd web && npx tsc -b --noEmit && npx eslint <touched files> && npx vitest run` passing.

```ts
import api from './api';

export type InstanceInfo = {
  mode: 'saas' | 'onprem';
  version: string;
  license: unknown;
};

let cached: Promise<InstanceInfo> | null = null;

/**
 * Fetches GET /api/instance once per page load and memoizes the result —
 * the deployment mode is a build/deploy-time value, never changes while a
 * tab is open. On failure, resolves to `{mode: 'onprem', ...}`: the same
 * safe default the backend itself uses for DEPLOYMENT_MODE, so a network
 * hiccup fails toward hiding saas-only surfaces, never toward exposing
 * them.
 */
export function getInstanceInfo(): Promise<InstanceInfo> {
  if (!cached) {
    cached = api
      .get('/api/instance')
      .then((res) => res.data as InstanceInfo)
      .catch(() => ({ mode: 'onprem' as const, version: 'unknown', license: null }));
  }
  return cached;
}
```

- [ ] **Step 7: Implement `useInstanceMode.ts`**

Create `web/src/hooks/useInstanceMode.ts`:

```ts
import { useEffect, useState } from 'react';
import { getInstanceInfo } from '@/lib/instance';

/**
 * Returns the current deployment mode, defaulting to 'onprem' (the safe
 * default) until the one-time GET /api/instance fetch resolves.
 */
export function useInstanceMode(): { mode: 'saas' | 'onprem'; loading: boolean } {
  const [mode, setMode] = useState<'saas' | 'onprem'>('onprem');
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    let cancelled = false;
    getInstanceInfo().then((info) => {
      if (!cancelled) {
        setMode(info.mode);
        setLoading(false);
      }
    });
    return () => {
      cancelled = true;
    };
  }, []);

  return { mode, loading };
}
```

```tsx
function ProtectedRoute({ children, requireSuperAdmin }: { children: JSX.Element, requireSuperAdmin?: boolean }) {
  const token = localStorage.getItem("token");
  if (!token) {
    return <Navigate to="/login" replace />;
  }
  
  if (requireSuperAdmin) {
    const user = JSON.parse(localStorage.getItem("user") || '{}');
    if (!user.is_super_admin) {
      return <Navigate to="/dashboard" replace />;
    }
  }
  
  return children;
}

function SaasOnlyRoute({ children }: { children: JSX.Element }) {
  const { mode, loading } = useInstanceMode();
  if (loading) {
    return null;
  }
  if (mode !== "saas") {
    return <Navigate to="/login" replace />;
  }
  return children;
}
```
