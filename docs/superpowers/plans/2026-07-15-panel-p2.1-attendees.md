## Board-reconciliation decisions (already made — implementers follow these)

1. **1g's top-tabs nav shell is the LOSING variant** — only its content region ships; the 1f rail (P1.2) stays the navigation. Rail's attendees step becomes the active link.
2. **Bulk bar "Print badges" → locked chip** ("coming with the badge editor", muted, not a button) until P3. Same honest-locked pattern as P1.2's rail rows.
3. **Bulk delete = sequential `DELETE /api/attendees/{id}`** over the (page-scoped, ≤50) selection with progress text; no bulk-delete endpoint. Typed-confirmation tier (board 1b lists "delete attendees (bulk)" as typed).
4. **Bulk-bar Export = client-side CSV of selection** (server export is whole-event only; see facts).
5. **Drawer "Regenerate code…" / "Delete…" = tier-1 destructive confirms** (typed tier reserved for bulk/event-wide ops). Regenerate = PATCH with client-generated `crypto.randomUUID()`.
6. **No "Full timeline →" link** (target page 3d isn't built). Activity shows up to 3 `MovementHistoryEntry` rows.
7. **Zone access is NOT materialized at import** — a category-like column maps to a custom field; dynamic access comes from P2.2 zone rules (evaluated at scan time). The board's "Map zone" error-row type does not exist.
8. **In-file duplicates (same non-empty email, case-insensitive) merge client-side keep-first at step 2**; duplicates against the EXISTING list come back from the server as per-row errors in step 3.
9. **Table Status cell:** plain text "Not checked in" (muted) / "Checked in" (success-colored text + check icon, WCAG-compliant) — the board only depicts the former; the latter derives from the StatusPill success vocabulary.
10. **Badge cell:** `printed_count > 0` → StatusPill "Printed" (ready/success); else "Not printed" (muted StatusPill with `status="empty"`).
11. **Encoding detection:** decode ArrayBuffer with `new TextDecoder("utf-8", { fatal: true })`; on throw → windows-1251. Additionally, if fatal-UTF-8 succeeds but the text has zero Cyrillic and the windows-1251 decode of the same bytes yields ≥30% more Cyrillic characters — prefer windows-1251 (catches valid-ASCII-but-mojibake edge). Override is a 2-way segmented control only.
12. **Wizard step 3 is not dismissable mid-import** (no ✕ while chunks are in flight); it becomes closable once all chunks settle.
