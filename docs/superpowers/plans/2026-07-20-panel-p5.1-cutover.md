# Panel P5.1 — Cutover mechanics — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a single combined nginx image that serves the new `panel/` SPA at `/` and the old `web/` app — reduced to the super-admin console only — under `/super-admin/`, and cut production over to it.

**Architecture:** `web/` is rebased (`base: '/super-admin/'`, router `basename`) and stripped to the console; a repo-root multi-stage `deploy/Dockerfile` builds both `panel/` and `web/` and copies their dists into one nginx (panel at html root, console under `super-admin/`); `deploy/nginx.conf` serves both with separate SPA fallbacks + immutable-asset + no-store-env.js blocks; prod-compose and CI point their single frontend image at the combined build. Panel and console share same-origin `localStorage` `token`/`operator_token`, so cross-SPA impersonation works unchanged.

**Tech Stack:** Vite + React (both SPAs), react-router (web/), nginx-unprivileged + envsubst, Docker multi-stage, docker-compose, GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-07-20-panel-p5.1-cutover-design.md` (user-approved).

## Global Constraints

- Branch `worktree-panel-p5.1-cutover` (worktree `.claude/worktrees/panel-p5.1-cutover`), base = main `05dd621`. Run from the worktree root; `git branch --show-current` before every commit (subagent-cwd hazard — see memory).
- **Topology unchanged**: still ONE frontend service in docker-compose (SaaS + on-prem); the combined image keeps the `…-web` name so compose refs and deploys are unchanged.
- `web/` after cutover = **console only**. Delete every customer-panel route + its now-orphaned page modules; keep the super-admin surface + the console's own login + the shared infra the console imports. Deletion is compiler-guided (unused-import/dead-file errors) — never delete anything the console still imports.
- Asset paths must not collide: panel `/assets/*`, console `/super-admin/assets/*`.
- Both SPAs get `PUBLIC_API_URL` at container start via `env.js.template` + envsubst; the console's env.js lands at `super-admin/env.js` (template subpath mirroring). `web/index.html` MUST change `<script src="/env.js">` → `<script src="%BASE_URL%env.js">`.
- Shared same-origin auth: `localStorage["token"]` + `localStorage["operator_token"]` are shared by panel + console; impersonation round-trip (console → panel banner) is a supported, tested flow.
- Backend, agent, and the Tauri `desktop/` kiosk are untouched. `panel/` + `packages/ui` are untouched (regression-guard their suites only).
- Panel typecheck is `npm run typecheck -w panel` (never bare tsc). web/ uses its own `npm run build`/`npm run test -w web` (or `cd web && npm test`).
- The combined-image build + staging run (Task 4) needs Docker; it is the controller/user-run phase-exit proof, not a subagent gate.

### Task 2: Combined `deploy/Dockerfile` + `deploy/nginx.conf`

**Files:**
- Create: `deploy/Dockerfile`, `deploy/nginx.conf`

**Interfaces:**
- Consumes: Task 1's rebased `web/` (built with `base: /super-admin/`) and the existing `panel/` build.
- Produces: an image serving panel at `/` + console at `/super-admin/`. Consumed by Task 3 (compose + CI reference it).

- [ ] **Step 1: Write `deploy/nginx.conf`** (console locations first for readability; longest-prefix match makes order immaterial, pinned by Task 4's routing test):

```nginx
server {
    listen 8080;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # Console SPA (old web/, rebased under /super-admin/)
    location /super-admin/ {
        try_files $uri $uri/ /super-admin/index.html;
    }
    location /super-admin/assets/ {
        add_header Cache-Control "public, max-age=31536000, immutable";
        try_files $uri =404;
    }
    location = /super-admin/env.js {
        add_header Cache-Control "no-store";
        try_files $uri =404;
    }

    # Panel SPA (new panel/) at root
    location / {
        try_files $uri $uri/ /index.html;
    }
    location /assets/ {
        add_header Cache-Control "public, max-age=31536000, immutable";
        try_files $uri =404;
    }
    location = /env.js {
        add_header Cache-Control "no-store";
        try_files $uri =404;
    }
}
```

- [ ] **Step 2: Write `deploy/Dockerfile`** (multi-stage; build context = repo root):

```dockerfile
# syntax=docker/dockerfile:1
# Combined P5 cutover image: panel/ at /, console (web/) at /super-admin/.
# Build context is the REPO ROOT (panel source-imports @idento/ui via npm
# workspaces; web/ is a standalone package).

# Stage 1 — panel (workspace build; needs packages/)
FROM node:24-alpine AS build-panel
WORKDIR /src
COPY package.json package-lock.json .npmrc ./
COPY packages ./packages
COPY panel ./panel
RUN npm ci && npm run build -w panel

# Stage 2 — console (web/ is NOT a workspace member: own package.json + lock)
FROM node:24-alpine AS build-web
WORKDIR /web
COPY web/package.json web/package-lock.json web/.npmrc ./
RUN npm ci
COPY web ./
RUN npm run build

# Stage 3 — one unprivileged nginx serving both
FROM nginxinc/nginx-unprivileged:1.27-alpine
USER root
COPY --chown=nginx:nginx --from=build-panel /src/panel/dist /usr/share/nginx/html
COPY --chown=nginx:nginx --from=build-web   /web/dist        /usr/share/nginx/html/super-admin
# Explicit chown: the base image's html dir pre-exists owned by root and
# COPY --chown only touches entries it creates, so the envsubst entrypoint
# (unprivileged UID 101) can't write env.js without this (see web/Dockerfile).
RUN chown nginx:nginx /usr/share/nginx/html /usr/share/nginx/html/super-admin
USER nginx
COPY deploy/nginx.conf /etc/nginx/conf.d/default.conf
# envsubst mirrors template subpaths into the output dir → the console's
# template lands at super-admin/env.js.
COPY panel/env.js.template /etc/nginx/templates/env.js.template
COPY web/env.js.template   /etc/nginx/templates/super-admin/env.js.template
ENV NGINX_ENVSUBST_OUTPUT_DIR=/usr/share/nginx/html
EXPOSE 8080
```

- [ ] **Step 3: Build the image locally** (Docker available on the controller/dev machine; if the executing subagent has no Docker, mark this step for controller run and proceed to commit the files):

```bash
docker build -f deploy/Dockerfile -t idento-web-combined:p51 .
```

Expected: build succeeds; both `npm run build`s complete. (The full run-and-curl verification is Task 4.)

- [ ] **Step 4: Commit**

```bash

git commit -m "deploy: combined nginx image — panel at /, console at /super-admin/"
```
